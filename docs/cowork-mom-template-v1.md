## B. Field semantics (parser contract)

### Metadata block

| Field | Required | Type | Notes |
|---|---|---|---|
| Judul Rapat | yes | string | Free text. Bukan source untuk task creation. |
| Tanggal | yes | date `YYYY-MM-DD` | Reference untuk dedup window + "MoM date". |
| Waktu | no | range `HH:MM-HH:MM WIB` | Reference only. Not used for parser decision. |
| Project Terkait | no | string | Default project untuk action items yang tidak override. Value `"Umum"` = no project preference. |
| Peserta | no | list of `Name <email>` | Reference only. Pemilik action item ada di per-action PIC. |

### Action item block

Heading format: `### [ACTION-XXX]` di mana `XXX` = 3-digit zero-padded sequence (001-999).

- **Unique within MoM file** — parser pakai untuk dedup.
- **Not globally unique** — different MoM files boleh pakai ACTION-001.
- **`source_file_id + action_id` composite** = global unique per task source (proposed di ADR-007).

| Field | Required | Type | Validation | Mapping ke `tasks` |
|---|---|---|---|---|
| Tugas | yes | string ≤200 char | Trim, non-empty | `title` |
| PIC | yes | `Nama <email>` | Email present, format `*@kalaborasi.com` | `assignee_id` (lookup via `users.email`) |
| Deadline | no | `YYYY-MM-DD` atau `TBD` | Parseable date kalau bukan TBD | `deadline` (NULL kalau TBD) |
| Prioritas | yes | enum | `Low|Medium|High|Urgent` | `priority` (lowercase) |
| Project | no | string atau `"Umum"` | Fallback ke metadata Project Terkait. `"Umum"` = no project | `project_id` (fuzzy lookup, auto-create kalau tidak ada) |
| Estimasi Jam | no | int atau `N/A` | Positive integer | `estimated_hours` (NULL kalau N/A) |
| Konteks | no | string ≤500 char | Free text | `description` (atau ditambah ke comment) |
| Status Awal | yes | enum | Selalu `Todo` | `status='todo'` (parser ignore other values per safety) |

### Optional sections

- `## CATATAN TAMBAHAN` — free text, not parsed (informational only).
- `## REFERENSI` — links recording + slides, capture as `task_sources.references jsonb` (optional).

## C. Parser robustness rules

1. **Header tolerant:** parser detect ACTION-XXX via regex `^### \[ACTION-(\d{3})\]` (case-sensitive). Whitespace trimmed.
2. **Field key tolerant:** lowercase + remove `: ` separator. `Deadline:` ≡ `deadline:` ≡ `Deadline :`.
3. **Empty value rejection:** required field empty → row marked `error`, included di summary tapi tidak di-create.
4. **PIC email validation:** Regex `[a-zA-Z0-9._%+-]+@kalaborasi\.com$` (domain locked Sprint 5 — pilot scope).
5. **Email lookup:** Match `users.email` exact. Tidak match → action `error` dengan reason `pic_not_found` (Q3 owner answer Sprint 4 = skip row pattern). Future Sprint 6+ dapat support fuzzy `users.full_name`.
6. **Project lookup:** Match `projects.name` case-insensitive. Tidak match + value bukan `"Umum"` → auto-create dengan `status='active'` (mirror F15 CSV import pattern).
7. **Status Awal locked:** Parser SET `status='todo'`. Even jika MoM tulis nilai lain, ignored (safety — Cowork tidak set non-default status di CREATE flow).

## D. Decision logic per action item

Per task brief Section "ACTION DECISION TABLE":

```
┌─────────────────────────────────────────────────────────────────────┐
│  Lookup task WHERE                                                  │
│    source_metadata->>'source_file_id' = {drive_file_id}             │
│    AND source_metadata->>'action_id' = '{ACTION-XXX}'               │
└─────────────────────────────────────────────────────────────────────┘
                              ↓
        ┌──────────────────── ↓ ─────────────────────┐
        ↓                     ↓                       ↓
   not found              found, fields              found, no
        ↓                 changed                     change
        ↓                     ↓                       ↓
   ┌────────┐           ┌────────────────┐      ┌──────────┐
   │ CREATE │           │ UPDATE         │      │ SKIP     │
   │ task   │           │ - emit comment │      │ log only │
   │ baru   │           │   "[Auto from  │      │          │
   │        │           │   MoM ...]"    │      │          │
   │        │           │ - status only  │      │          │
   │        │           │   change kalau │      │          │
   │        │           │   keyword di   │      │          │
   │        │           │   Konteks      │      │          │
   │        │           │   match        │      │          │
   │        │           │   (selesai/    │      │          │
   │        │           │   blocked/     │      │          │
   │        │           │   review)      │      │          │
   └────────┘           └────────────────┘      └──────────┘
```

**Status keyword detection (UPDATE flow):**
- `selesai|done|completed` → status='done'
- `blocked|stuck|menunggu` → status='blocked'
- `review|cek dulu` → status='review'
- otherwise → no status change

## F. Mutability + future versioning

- **V1 = Sprint 5 baseline.** Real Plaud output bisa berbeda; parser harus gracefully reject/warn malformed sections (bukan crash).
- **V2 candidates** (post-Sprint 5 validation):
  - Multi-language MoM (English variant)
  - Recurring task hint (`Frekuensi: Weekly|Monthly`)
  - Subtask hierarchy via nested `[ACTION-001-A]` sub-IDs
- **Backward compat:** V2 parser harus tetap accept V1 format (versioning di metadata `MoM Version: v1`).
